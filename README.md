Social Media Engagement Analytics Pipeline
Project Overview

This project builds a data analysis pipeline for social media engagement data using Python, Pandas, and MySQL.

The system processes raw social media data, performs feature engineering, engagement analytics, sentiment analysis, and time-based analysis, and stores the cleaned dataset in a MySQL database for further querying and reporting.

The goal is to identify patterns in user engagement based on:

Post type

Posting time

Platform

Sentiment

This type of analysis is commonly used by digital marketing teams and data analysts to optimize social media strategies.

Dataset Description

The dataset contains information about social media posts and their engagement metrics.

Column	Description
post_time	Timestamp when the post was published
post_day	Day of the week
platform	Social media platform
post_type	Type of content (image, video, text, etc.)
likes	Number of likes
comments	Number of comments
shares	Number of shares
sentiment_score	Sentiment label (positive, negative, neutral)
Data Processing Workflow
1 Data Ingestion

The dataset is loaded using Pandas.

df = pd.read_csv("social_media.csv")
2 Feature Engineering
Total Engagement

A new feature is created to calculate overall engagement.

Total Engagement = likes + comments + shares
df['Total_Engagement'] = df[['likes','comments','shares']].sum(axis=1)
Post Hour Extraction

Extract the hour from the timestamp.

df['post_hour'] = df['post_time'].dt.hour
Engagement Rate

Engagement rate is calculated assuming 1000 followers.

Engagement Rate = Total Engagement / 1000
Average Engagement by Post Type

Used to identify which content format performs best.

df.groupby('post_type')['Engagement_Rate']
3 Best Day Analysis

Finds the day that produces the highest average engagement.

df.groupby('post_day')['Total_Engagement'].mean()
4 Top Performing Content

Calculates the total engagement generated by each post type.

df.groupby('post_type')['Total_Engagement'].sum()
5 Posting Time Analysis

Posts are categorized into time segments:

Time Range	Category
5 – 12	Morning
12 – 17	Afternoon
17 – 21	Evening
Other	Night

This helps determine the best time to publish posts.

6 Engagement Summary Tables

Multiple aggregated reports are generated.

Post Type Summary

Average engagement metrics for each content type.

Time-Based Summary

Engagement patterns based on posting time.

Platform + Post Type Summary

Cross-platform engagement analysis.

7 Sentiment Analysis

Sentiment labels are converted to numeric values.

Sentiment	Numeric Value
Positive	1
Neutral	0
Negative	-1

Correlation is then calculated between:

Sentiment Score vs Total Engagement

This helps evaluate whether positive posts drive higher engagement.

8 Data Export

Processed data is exported to CSV files.

File	Description
cleaned_data.csv	Final cleaned dataset
summary_day.csv	Aggregated engagement report
9 Database Integration

The cleaned dataset is stored in MySQL using SQLAlchemy.

df.to_sql("cleaned_social", con=engine, if_exists="replace")

Database Details

Database : social
Table    : cleaned_social

Data can then be queried using SQL.

Example:

SELECT Total_Engagement
FROM cleaned_social;
